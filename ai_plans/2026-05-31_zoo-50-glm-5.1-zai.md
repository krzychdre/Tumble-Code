# Zoo PR #50 ‚Äî Add GLM-5.1 to the Z.AI provider

**Upstream:** Zoo-Code PR #50, squash-merged 2026-05-10 (`5a5b22ed5`)
**Branch:** `feature/zoo-50-glm-5.1-zai` (off `main`)
**Type:** Feature ‚Äî new model + output-token clamping fix + tests.

## What the upstream PR does

1. **`packages/types/src/providers/zai.ts`** ‚Äî add a `glm-5.1` entry to both
   `internationalZAiModels` (200k ctx, 131,072 max out, in $1.4 / out $4.4, cacheReads
   $0.26) and `mainlandZAiModels` (204,800 ctx, 131,072 max out, in $0.68 / out $2.28,
   cacheReads $0.13); both `supportsReasoningEffort: ["disable","medium"]`,
   `reasoningEffort: "medium"`, `preserveReasoning: true`, `supportsPromptCache: true`.
   Plus a doc-link comment. Model-id unions are `keyof typeof ‚Ä¶`, so they auto-extend.
2. **`src/shared/api.ts`** ‚Äî `getModelMaxOutputTokens` adds an `isZaiProvider`
   (`settings?.apiProvider === "zai"`) bypass alongside the existing `isGpt5Model` one, so
   Z.ai's hand-curated `maxTokens` (e.g. 128k for glm-5.1) is not clamped to 20% of the
   context window. Serves callers that pass full `ProviderSettings` (Task context mgmt,
   model-params).
3. **`src/api/providers/zai.ts`** ‚Äî `createStreamWithThinking` stops calling
   `getModelMaxOutputTokens` (which can't see `apiProvider` because `ApiHandlerOptions`
   omits it) and uses `this.options.modelMaxTokens || (info.maxTokens ?? undefined)`
   directly, with an explanatory comment.
4. **Unit tests** ‚Äî `src/api/providers/__tests__/zai.spec.ts` (+36) and
   `src/shared/__tests__/api.spec.ts` (+36) cover the new model and the clamp bypass.
5. **e2e** ‚Äî `apps/vscode-e2e/src/suite/providers/zai.test.ts` (+214): a self-contained
   suite using an inline `globalThis.fetch` interceptor (passthrough when `ZAI_API_KEY`
   set, mocked otherwise) asserting GLM-5.1 output-token behavior.
6. **`apps/vscode-e2e/AGENTS.md`** (+19) ‚Äî docs for the `@copilotkit/aimock` fixture
   workflow.
