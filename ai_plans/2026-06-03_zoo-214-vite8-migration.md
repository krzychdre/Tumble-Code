## §1 What it does

Migrates the webview build from Vite 6 (Rollup) to Vite 8 (Rolldown/Oxc):

- `vite 6.3.6 → ^8.0.13`, `@vitejs/plugin-react ^4.3.4 → ^5.2.0`,
  `@types/node 20.x → ^20.19.25` (webview-ui and apps/vscode-e2e).
- `resolve.alias` (manual `@`/`@src`/`@roo`) → `resolve.tsconfigPaths: true`
  (Vite 8 reads the path mappings already declared in `webview-ui/tsconfig.json`).
- `build.rollupOptions` → `build.rolldownOptions`; `input` object → single string;
  simplified `chunkFileNames`/`assetFileNames`; dropped the manual `mermaid-bundle`
  `manualChunks` and the `optimizeDeps.include` for mermaid/dagre (Rolldown's
  default code-splitting handles these).
- `minify: mode === "production" ? "esbuild" : false` → `minify: mode === "production"`
  (boolean; Rolldown/Oxc default minifier).

## §2 Scope cuts / landmines

- Build-tooling only — no product code, no behavior, no i18n, no branding.
- Kept our fork-specific `define` block verbatim (`PKG_OUTPUT_CHANNEL` = "Roo-Code")
  — internal id stays; do NOT rebrand here.
- Our `apps/vscode-e2e/package.json` does **not** carry `@copilotkit/aimock`
  (the fork rejected that family); only the `@types/node` bump applies there.
- `@tailwindcss/vite@4.1.6` emits a peer warning (wants vite ^5/^6, got 8) — it is
  **non-fatal**: the production build still emits the single combined
  `assets/index.css` (verified, 140 KB).
